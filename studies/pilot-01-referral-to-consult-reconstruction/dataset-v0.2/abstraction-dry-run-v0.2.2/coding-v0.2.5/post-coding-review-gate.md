# Post-Coding Review Gate

Status: `closed pending post-coding review`

This gate controls whether Dataset v0.2 work may proceed beyond the two-case limited dry-run coding pass.

## Current Scope

Coding has been completed only for:

- `ABS-001`
- `ABS-002`

The gate does not authorize:

- coding any additional cases;
- abstracting the remaining 8 candidates;
- treating v0.2.5 results as clinical validation;
- using any output as clinical guidance;
- changing the original abstractions.

## Review Required Before Scaling

A reviewer should inspect:

- [ABS-001 coding](case-coding/ABS-001-coding.md)
- [ABS-002 coding](case-coding/ABS-002-coding.md)
- [Results table](results-table.md)

## Gate Decision Options

Choose one after review:

- [x] closed pending post-coding review
- [ ] open for second-coder repeat of ABS-001 and ABS-002 only
- [ ] open for revision of ABS-001 / ABS-002 coding only
- [ ] open for abstraction of a small next batch
- [ ] remain closed; revise abstraction method before continuing

## Required Decision Record

Before any next step, document:

- reviewer(s);
- review date;
- whether coding outputs were interpretable;
- whether coding stayed source-distant and non-guidance;
- whether rubric scores were useful;
- whether a second coder should repeat the dry run;
- whether the remaining 8 candidates remain on hold or move to a limited next batch.

## Default Next Step

The recommended default is a post-coding review before any second-coder work or further abstraction.

# Abstraction Safeguards Checklist

Use this checklist before, during, and after every Dataset v0.2.1 abstraction. The purpose is to create paraphrased educational artifacts for methodology testing, not clinical documentation.

## Before Abstraction

- [ ] Confirm the source is public.
- [ ] Confirm no paywalled text is being copied.
- [ ] Confirm no source narrative will be reproduced.
- [ ] Confirm no patient identifiers will be retained.
- [ ] Confirm rare-case identifiability risk.
- [ ] Confirm the educational abstraction label will be included.
- [ ] Confirm citation metadata will be stored separately from generated artifact pairs.
- [ ] Confirm the abstraction is not real clinical documentation.

## During Abstraction

- [ ] Paraphrase at high level using original research-team wording.
- [ ] Preserve the cognitive transition only.
- [ ] Avoid copying timeline details unnecessarily.
- [ ] Avoid copying unique narrative phrasing.
- [ ] Avoid specific geographic or institutional identifiers.
- [ ] Avoid dates, initials, record numbers, or exact demographic combinations.
- [ ] Preserve uncertainty only as needed for methodology.

## After Abstraction

- [ ] Verify no copied source text.
- [ ] Verify no patient-identifying information.
- [ ] Verify the abstraction is not clinical guidance.
- [ ] Verify source citation remains separate.
- [ ] Verify the artifact pair is labeled educational abstraction.
- [ ] Verify a second reviewer can inspect the abstraction before coding.

## Stop Condition

Stop abstraction if the case cannot be represented without copied narrative, source images, source tables, rare identifying detail, or clinical advice.

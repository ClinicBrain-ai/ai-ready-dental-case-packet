# Pre-Abstraction Review Memo

This memo records the transition from Dataset v0.2 source selection to Dataset v0.2.1 limited abstraction planning. No case abstraction has been performed.

## 1. Why Abstraction Should Not Begin Until the Revised Shortlist Is Accepted

Abstraction should begin only after the revised shortlist is accepted because the original shortlist included one candidate with elevated identifiability risk. A clear v0.2.1 shortlist prevents abstractors from working from competing candidate sets and makes the first abstraction batch auditable.

## 2. Why CAND-014 Was Replaced With CAND-016

CAND-014 was methodologically valuable but involved a rare-disease presentation with higher indirect identifiability risk. CAND-016 preserves high cognitive-transition value while offering a more manageable first-batch risk profile. The replacement supports a conservative first move from synthetic cases toward public educational sources.

## 3. Whether v0.2.1 Is Ready for Limited Abstraction

Dataset v0.2.1 is ready for limited, citation-linked, paraphrased educational abstraction. It is not ready for coding because no artifact pairs have been created or reviewed.

Abstraction should proceed slowly, one candidate at a time, using `abstraction-safeguards-checklist.md` and `abstraction-output-template.md`.

## 4. What Risks Remain

Remaining risks include:

- accidental copying of source narrative;
- reproduction of image-dependent findings;
- retention of rare identifying details;
- over-specific clinical chronology;
- clinical-guidance tone in downstream plan language;
- abstraction drift away from the original educational source pattern.

## 5. What Safeguards Must Be Followed

Every abstraction must:

- use original research-team wording;
- retain citation metadata separately;
- omit names, dates, institutions, locations, initials, record numbers, and unnecessary exact demographics;
- avoid source figures, tables, captions, and unique phrasing;
- include an educational-use disclaimer;
- undergo second-review inspection before coding.

## 6. Whether Research Note 002 Should Wait Until Coding and Reconciliation Are Complete

Research Note 002 should wait until abstraction, two-coder coding, and reconciliation are complete. A source-selection or abstraction-only note would be premature because it would not test whether the reconstruction workflow produces interpretable coding outputs on v0.2.1.

## 7. Whether a Human Reviewer Should Inspect Abstractions Before Coding

Yes. A human reviewer should inspect every abstraction before coding. The reviewer should verify that the artifact pair is educational, paraphrased, non-identifying, non-guidance, and sufficiently structured for independent reconstruction coding.

## Pre-Abstraction Decision

Proceed with v0.2.1 limited abstraction only under safeguards. Do not publish or code any abstraction until second-review inspection is complete.

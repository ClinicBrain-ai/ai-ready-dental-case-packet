# Coding Framework

This framework defines coding categories for referral-to-consult cognition reconstruction. A single artifact transition may receive multiple codes.

## Preservation

Definition: Cognition remains traceable across the referral and consultation artifacts.

Example: The referral concern of unexplained oral lesion remains the central concern in the specialist note.

Coding rule: Use when the same claim, concern, uncertainty, treatment intent, or evidence priority appears in both artifacts with substantially similar meaning.

## Compression

Definition: Complex or uncertain cognition is reduced to a shorter, more definitive, or less detailed representation.

Example: A broad referral differential becomes a single diagnosis in the consultation note without explicit discussion of alternatives.

Coding rule: Use when downstream documentation preserves a conclusion but reduces visible rationale, alternatives, or uncertainty.

## Delegation

Definition: Cognitive work is assigned to the specialist, another actor, an AI system, or a workflow.

Example: The referral states, "please determine whether biopsy is indicated," delegating risk interpretation and action planning.

Coding rule: Use when an artifact explicitly or implicitly transfers interpretation, decision-making, uncertainty resolution, or planning to another participant.

## Translation

Definition: Cognition changes language, audience, or operational form while preserving core meaning.

Example: Specialist reasoning is translated into patient-facing follow-up instructions.

Coding rule: Use when the cognitive content appears adapted for a different audience or workflow without changing the core clinical meaning.

## Reframing

Definition: The organizing clinical problem changes across the transition.

Example: A referral framed as infection is reframed by the specialist as autoimmune disease.

Coding rule: Use when the downstream artifact changes the primary diagnostic, causal, or management frame.

## Diagnostic Replacement

Definition: A suspected or working diagnosis is replaced by a different diagnosis downstream.

Example: Referral suspects abscess; consultation concludes malignancy is the primary concern.

Coding rule: Use when a downstream diagnosis displaces the source diagnosis rather than merely adding to it.

## Mechanism Substitution

Definition: The causal explanation changes while the clinical phenomenon may remain similar.

Example: Pain attributed upstream to infection is attributed downstream to nerve injury.

Coding rule: Use when the downstream artifact explains the same finding through a different mechanism.

## Uncertainty Loss

Definition: Uncertainty present or implied upstream becomes less visible downstream.

Example: A referral asks whether a lesion is traumatic or neoplastic; the consultation documents a plan without preserving the original uncertainty.

Coding rule: Use when uncertainty markers, alternatives, caveats, or unknowns are omitted or converted into confident statements.

## New Cognition

Definition: New claims, diagnoses, mechanisms, risks, or recommendations appear downstream without a clear upstream source.

Example: The specialist introduces a systemic disease concern not mentioned in the referral.

Coding rule: Use when downstream cognition cannot be traced to the referral artifact or provided metadata.

## Missing Cognition

Definition: Reasoning needed to understand the transition is absent or unrecoverable.

Example: The referral suspects infection but includes no findings supporting that suspicion.

Coding rule: Use when the artifact pair leaves a clinically important reasoning gap that cannot be filled without speculation.

## Coding Notes

- Prefer multiple precise codes over one broad code.
- Preserve uncertainty when coders disagree.
- Do not infer private thought beyond what the artifacts can support.
- Record short evidence quotes or paraphrases for each code.

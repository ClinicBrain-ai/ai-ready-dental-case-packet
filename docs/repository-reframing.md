# Repository Reframing

This repository is now framed under the Clinical Cognition Transformation Lab (CCTL).

CRTL, the Clinical Reasoning Transformation Lab, was an intermediate framing. It helped move the repository beyond dental AI infrastructure and DCS, but it remained centered on clinical reasoning. CCTL is the broader parent identity.

## Current Parent Identity

Clinical Cognition Transformation Lab (CCTL)

- Clinical Reasoning Transformation
- Human-AI Co-Cognition
- Cognitive Provenance
- Cognitive Reconstruction
- Longitudinal Cognitive Change
- Future Clinical Expertise

## Primary Research Object

The primary research object is Human-AI Clinical Cognition.

Human-AI Clinical Cognition refers to the distributed cognitive processes that emerge when clinical decisions are jointly produced by clinicians, AI systems, documentation artifacts, institutional workflows, healthcare organizations, patients, and caregivers.

## Core Thesis

Clinical cognition is no longer exclusively an individual cognitive activity. It is increasingly a distributed process emerging from interactions among clinicians, AI systems, documentation artifacts, institutions, and patients.

The scientific challenge is no longer understanding clinical reasoning alone. The scientific challenge is understanding how clinical cognition transforms within distributed human-AI systems.

## What This Repository Is No Longer

This repository is no longer primarily:

- a dental AI tooling project;
- a CBCT conversion project;
- an interoperability project;
- an ontology design project;
- a standards project;
- an AI-ready dental packet project;
- a software infrastructure project alone.

Those remain historical origins or downstream artifacts.

## Project Evolution

-> Clinical Cognition Transformation

## Role of Dental and DCS Materials

The dental packet and dental imaging work were early domain-specific infrastructure experiments that helped reveal the broader research question.

DCS is now a possible cognitive provenance representation layer. It may help encode clinical context, reasoning traces, uncertainty, transformations, and workflow transitions, but it is not the central research question.

The existing DCS files, schemas, examples, governance notes, and dental case materials should be preserved as historical and experimental infrastructure. They show how the project reached the CCTL framing and may still support downstream representation studies.

## Reframing Rule

When describing this repository publicly, lead with CCTL and Human-AI Clinical Cognition.

When describing DCS, present it as:

A candidate cognitive provenance representation layer that emerged from early dental AI infrastructure work.

Do not present DCS as the central mission, the primary research question, or the final scientific object of the repository.
